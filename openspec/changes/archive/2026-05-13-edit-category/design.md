## Context

当前系统使用 NestJS + MongoDB 后端和 UniApp + Vue3 前端。Market（分类）数据存储在 MongoDB 中，每个分类包含 name、image 和 foods 数组。现有 API 支持创建、删除分类和增删改菜品，但不支持修改分类信息。

前端分类图片目前是静态资源（`/static/...`），需要改为后端动态管理。

## Goals / Non-Goals

**Goals:**
- 支持修改分类名称（name）和图片（image）
- 名称唯一性校验
- 图片上传复用现有接口
- 前端编辑按钮仅在编辑模式显示
- 逐步迁移现有静态图片

**Non-Goals:**
- 删除分类功能（已有接口，前端暂不添加）
- 批量编辑分类
- 分类排序调整

## Decisions

**1. 使用现有 uploadImage 接口处理图片上传**
- 理由：避免重复代码，保持上传逻辑一致性
- 替代方案：新建分类专用上传接口（增加维护成本）

**2. 编辑按钮放在左侧分类列表**
- 理由：与分类项关联直观，点击后直接知道编辑哪个分类
- 替代方案：右侧标题栏（需要额外标识当前分类）

**3. PUT 请求体使用 `{ id, name, image }` 格式**
- 理由：与现有 updateFood 接口风格一致
- id 使用 MongoDB 的 _id 字段

**4. 重名校验逻辑**
- 更新时排除自身，检查其他文档是否已存在该名称
- 存在则返回 409 Conflict

## Risks / Trade-offs

**[风险] 现有分类图片是静态资源，需要逐步迁移**
→ 缓解：编辑时重新上传图片，未编辑的分类保持现状

**[风险] 前端同时显示编辑按钮可能误触**
→ 缓解：编辑按钮仅在编辑模式（currentType=1）显示，使用小图标

**[风险] 并发更新可能导致重名检查失效**
→ 缓解：添加数据库唯一索引（建议后续优化）
